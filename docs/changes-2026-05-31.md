# 变更记录 — 2026-05-31

## 1. 消息 ID 生成策略优化
- **文件**: `frontend/src/types/index.ts`
- **变更**: `nextMsgId()` 改用 `crypto.randomUUID().slice(0, 8)` 替代 `Date.now()`
- **原因**: 避免快速连续发送消息时 `Date.now()` 可能产生的 ID 冲突
- **影响范围**: 所有使用 `nextMsgId()` 的消息创建（用户消息、助手回复、错误消息、进度消息）

## 2. Sidebar 组件原子化拆分
- **变更**: 
  - 创建 `frontend/src/components/Sidebar/SessionItem.tsx` — 会话条目子组件（已含 `React.memo`）
  - 创建 `frontend/src/components/Sidebar/DbTableNode.tsx` — 数据库表树节点子组件（新增 `React.memo`）
  - `Sidebar.tsx` 改为从新文件导入子组件
- **原因**: 提升代码可维护性，方便独立测试

## 3. Sidebar 回调函数稳定性优化
- **变更**: 
  - 导入 `useCallback` 并用 `handleRequestDelete` 包裹 `setConfirmDeleteId`
  - 移除了内联箭头函数 `(id) => setConfirmDeleteId(id)` 
- **原因**: 内联回调每次渲染创建新引用，破坏 `SessionItem`（React.memo）的重渲染跳过优化

## 4. TS 清理
- **变更**: 移除 `Sidebar.tsx` 中未使用的 `useRef` 和未调用的辅助函数
- **结果**: `npx tsc --noEmit` 零错误

## 5. 全量 Store Selector 模式推行
- **涉及文件** (6 个):
  - `DatabaseManagerPage.tsx` — `databases`, `setDatabases`
  - `LoginPage.tsx` — `login`, `isLoading`
  - `RegisterPage.tsx` — `register`, `isLoading`
  - `Dashboard.tsx` — `dashboardItems`, `removeDashboardItem`, `setDashboardItems`
  - `MarkdownRenderer.tsx` (QueryBadge) — `addDashboardItem`, `dashboardItems`, `removeDashboardItem`
  - `App.tsx` — `user`, `isInitialized`
- **变更**: `const { a, b } = useStore()` → `const a = useStore(s => s.a); const b = useStore(s => s.b)`
- **原因**: 解构订阅会使组件在 Store 中任何字段变化时都重渲染；Selector 模式确保只在依赖字段变化时重渲染
- **结果**: 至此项目中所有 Store 订阅均采用 Selector 模式，零解构订阅残留

## 6. SSE 增强评估
- **结论**: 当前 SSE 实现（`useChatStream.ts`）为一请求一流模式，`@microsoft/fetch-event-source` 的自动重连在该模式下不适用（断开后后端分析状态丢失），不引入新依赖
- **替代方案**: 现有实现已通过 35 项测试验证，运行稳定
