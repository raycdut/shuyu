📊 Request Flow Comparison

Shuyu: without RAG (current) vs. with RAG (proposed)

❌ Without RAG (Current)

Step 1
User asks a question
"上个月华东区销量排名"
▼
Step 2
Load full schema from SQLite
load_full_schema() SELECT * FROM imported_tables WHERE database_id=?
All 30+ tables, 450+ columns loaded into memory
▼
Step 3
Build schema prompt
build_schema_prompt() Format every table + column + description
~3,000-5,000 tokens of schema text
▼
Step 4
Inject into agent system prompt
routes/chat.py:203
Agent sees ALL tables, must infer which are relevant
▼
Step 5
SQL generation
sql_tool.py LLM reads full schema → generates SQL
⚠️ Risk: LLM picks wrong table (e.g. "suppliers" has a "region" column)
▼
Step 6
Execute SQL → return result
connector.execute() → formatted response
Cost per query
~4,000 tokens for schema alone
Plus LLM generation. Schema-to-response ratio: ~50% waste

✅ With RAG (Proposed)

Step 1
User asks a question
"上个月华东区销量排名"
▼
Step 2
Embed question
embed() user_question → vector (1536 dims)
~100ms, via OpenAI text-embedding-3-small
▼
Step 3
Vector search → top-K tables
schema_retriever.retrieve()
· orders: score 0.91 — "销量" ✓
· products: score 0.68 — "排名→产品" ✓
· regions: score 0.52 — "华东区" ✓
· suppliers: score 0.12 — ✗ filtered
· inventory: score 0.09 — ✗ filtered
▼
Step 4
Load relevant schema only
load_imported_columns() × 3 tables
~500-800 tokens → 80% less than full schema
▼
Step 5
SQL generation (cleaner prompt)
sql_tool.py LLM sees only relevant schema
✅ Less noise, less hallucination risk, faster generation
▼
Step 6
Execute SQL → return result
connector.execute() → formatted response
Cost per query
~700 tokens for schema + ~$0.00004 embedding
80-90% schema token reduction. Embedding cost is negligible.

Key Observations

  • The RAG path adds one extra step (embedding + vector search) but removes 80-90% of schema tokens
  • Embedding latency (~100ms) is far less than the time saved from a shorter prompt
  • Table selection is guaranteed by semantic similarity, not LLM guessing
  • Fallback path: if embedding service is down, inject full schema as-is
  • No changes needed to SQL generation, agent loops, or execution pipeline
