🌐 Entry Layer

Routes + lifespan + CORS
UNCHANGED POST /api/chat + /api/chat/stream (routes/chat.py)
Session, DB selection, schema injection
▼
📂 Data Layer
UNCHANGED ConfigDB (configdb/)
SQLite/MySQL — stores prompts, users, sessions
UNCHANGED Schema Persistence (persistence/schema.py)
ImportedTable + ImportedColumn ORM
N/A Vector Store
❌ Does not exist — schema loaded from SQLite directly
▼
🧠 Schema Processing
UNCHANGED build_schema_prompt() (db/schema.py)
Formats ALL tables + columns into prompt text
N/A Schema Router / Retriever
❌ Does not exist — full schema always injected
N/A Embedding Service
❌ Does not exist — no embedding calls made
▼
🤖 Agent Layer
UNCHANGED SimpleAgent (fast mode)
ReAct loop — system prompt contains ALL schema
UNCHANGED AdvancedAgent (quality mode)
Plan → Reflect → Execute — schema in plan prompt + exec prompt
UNCHANGED SQL Generation Tool (sql_tool.py)
LLM gets full schema_prompt → generate SQL
▼
🗄️ Database Layer
UNCHANGED DuckDB / MySQL / PostgreSQL
Remote database being queried
⚠️ Scaling Constraints
  • Full schema injected every query → token waste
  • 30 tables ≈ 5K tokens of schema prompt
  • 50+ tables → LLM starts confusing similar column names
  • No semantic table selection — relies on LLM inference
  • Multi-DB impossible — one schema per injection

✅ Proposed Architecture (With RAG)

🌐 Entry Layer
UNCHANGED FastAPI Server (main.py)
CHANGED POST /api/chat (routes/chat.py)
Calls schema_retriever.retrieve() instead of build_schema_prompt()
▼
📂 Data Layer
UNCHANGED ConfigDB (configdb/)
UNCHANGED Schema Persistence (persistence/schema.py)
NEW Vector Store (persistence/vector_store.py)
Table + column embeddings stored in SQLite-backed vector index (numpy cosine similarity)
▼
🧠 Schema Processing
CHANGED build_schema_prompt() (db/schema.py)
Still exists but now receives TOP-K tables only
NEW Schema Retriever (router/schema_retriever.py)
embed → vector search → format top-K schema
NEW Embedding Service (embedding/service.py)
Pluggable: OpenAI | SiliconFlow | local sentence-transformers
▼
🤖 Agent Layer
UNCHANGED SimpleAgent (fast mode)
ReAct loop — system prompt now has RELEVANT schema only
UNCHANGED AdvancedAgent (quality mode)
Same plan/exec/report pipeline with smaller schema prompt
UNCHANGED SQL Generation Tool (sql_tool.py)
Same code — gets smaller schema_prompt, less noise
▼
🗄️ Database Layer
UNCHANGED DuckDB / MySQL / PostgreSQL
✅ Key Improvements
  • Only top-K relevant tables injected → 80-90% token reduction
  • Scalable to 100+ tables without prompt budget issues
  • Multi-DB: can retrieve from N vector indexes independently
  • Embedding cost negligible (~$0.00004/query)
  • Zero changes to agent loops or SQL execution pipeline
  • Toggle ON/OFF via config — safe rollback
