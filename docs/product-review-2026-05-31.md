## P0 — 立刻做

### 1. Chat 中直接渲染表格 + 简单图表

查询结果目前只返回文本。应该在对话气泡里直接渲染表格（HTML table），少量数据可附带简易柱状图。

**优先级最高的体验断点。** 这比新增任何功能都重要。

### 2. 默认页改成系统概况

用户登录后看到空聊天界面不知道做什么。应该默认显示：
- 已连接数据库数
- 可用表的数量
- 最近对话列表
- 快速开始提示

### 3. README 只列已实现的功能

目前写了 12 种数据库但只 DuckDB 能用。改成：
- ✅ DuckDB (Working)
- 🔜 SQLite, PostgreSQL 开发中
- 🔮 更多规划中

诚实比野心更能赢得早期用户信任。

---

## P1 — 尽快做

### 4. 在线 Demo (Playground)

开源项目被发现的#1转化渠道。
- 部署一个在线 demo
- 带示例 DuckDB 数据
- 用户打开就能问「上月销售额多少」
- 体验完想用才去 docker compose

### 5. 降低 Onboarding 步骤

当前流程：
```
Docker → 配置 LLM → 配置 DB → 开始问（5 步）
```

目标流程：
```
Docker → 预置示例数据 + 免费 DeepSeek key → 直接问（2 步）
```

核心：开箱即用，10 秒内看到第一条回答。

### 6. README 顶部加截图/GIF

现在 README 纯文字。放一张聊天界面截图 + 简短 demo GIF，比一千字描述都有用。

## P2 — 有了更好

### 7. GitHub Releases + Changelog

打 v0.1.0 tag，写清楚 "what works"。让关注者知道项目还活着。

### 8. Contributing Guide + Issue Templates

降低他人参与门槛。一个人能写的代码有限，开源的力量在协作。

### 9. 英文 FAQ / Troubleshooting

解决常见问题：连不上数据库、API key 在哪配、Docker 端口冲突等。

## ❌ 不建议做的

- ~~Dashboard/看板~~ — 已决定删
- ~~更多数据库连接器~~ — 除非有用户明确要求，否则做了也没人用
- ~~权限系统/RBAC~~ — 单机部署不需要

## 一句话总结

> 差距不在功能数量，在 **用户体验完整度**。
> 修好「查询结果可视化」和「开箱即用」，项目就从"能跑的 demo"变成"想用的产品"。
