# 前端架构优化与功能扩展计划

本文档旨在梳理当前前端架构的瓶颈，并制定后续功能扩展（如可视化、看板、多 Agent 交互）的优化路径。

---

## 1. 架构层面优化 (Architecture)

### 1.1 引入轻量级状态管理 (Zustand)
**现状**: 所有状态堆积在 `App.tsx`，Prop Drilling 严重。
**方案**: 
- 建立 `useStore`，将全局共享状态（LLM 配置、数据库连接状态、当前会话 ID）移入 store。
- 减少组件间的参数传递，提升渲染效率。

### 1.2 业务逻辑解耦 (Custom Hooks)
**现状**: `App.tsx` 承载了复杂的 SSE 解析和 API 调用逻辑。
**方案**:
- 抽离 `useChatStream`: 专门负责 SSE 的连接、缓冲区处理和进度状态更新。
- 抽离 `useSessions`: 负责会话的增删改查逻辑。

### 1.3 强化 Markdown 渲染引擎
**现状**: 手写正则解析，难以支持复杂语法。
**方案**:
- 引入 `react-markdown` 配合 `remark-gfm`（表格支持）。
- 引入 `react-syntax-highlighter`: 为 SQL 和代码块提供更好的高亮体验。

---

## 2. 功能扩展规划 (New Features)

### 2.1 数据可视化 (Data Visualization)
**目标**: 让 Agent 返回的数据不只是表格，还能自动生成图表。
**路径**:
- 引入 **Recharts** 或 **ECharts**。
- **智能识别**: 当查询结果包含“时间+数值”或“分类+数值”时，自动渲染折线图或柱状图。
- **手动切换**: 在消息气泡中提供“表格/图表”切换按钮。

### 2.2 数据看板 (Dashboard Mode)
**目标**: 允许用户将重要的查询结果固定到看板。
**路径**:
- 在 `MessageBubble` 中增加“固定到看板”按钮。
- 新增 `Dashboard` 页面/面板，采用网格布局展示已固定的查询结果。

### 2.3 文件与上下文扩展
**目标**: 支持上传 CSV/Excel 进行即时分析。
**路径**:
- 增加文件上传组件。
- 后端对接 DuckDB 的文件读取能力，将文件转化为临时表进行分析。

---

## 3. 用户体验增强 (UX/UI)

### 3.1 虚拟滚动 (Virtualized List)
- **场景**: 当历史会话非常长（几百条消息）时。
- **方案**: 引入 `react-window`，只渲染当前可视区域的消息，保持页面丝滑。

### 3.2 响应式细节优化
- 适配移动端布局，侧边栏在小屏下自动折叠为抽屉。
- 增加“深色模式”支持（遵循系统偏好）。

---

## 4. 实施路线图

1. **Phase 1 (基础重构)**: 引入 Zustand，将 `App.tsx` 的配置和会话逻辑抽离。
2. **Phase 2 (插件化)**: 更换 Markdown 渲染引擎，支持更标准的表格和代码高亮。
3. **Phase 3 (能力增强)**: 集成 Recharts，实现基础的自动化图表展示。
4. **Phase 4 (场景扩展)**: 开发 Dashboard 看板功能。
